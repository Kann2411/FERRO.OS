# 04 - MODULES

> "Every module tells a story."

---

# Overview

FERRO.OS is composed of independent modules.

Each module represents one part of Kristian Ferrin's professional identity.

Modules are not pages.

Modules are applications inside the operating system.

Every module has:

• A purpose

• A personality

• A visual identity

• Unlock conditions

• Exploration rewards

• Sounds

• Animations

• Connections with FERRO CORE

---

# Module Categories

FERRO.OS contains four major categories.

## System Modules

Responsible for operating the OS.

# Module Rules

Every module must:

✓ Have a purpose.

✓ Be visually unique.

✓ Reward exploration.

✓ Feel like a real application.

✓ Be independently maintainable.

✓ Respect the design system.

✓ Tell part of Kristian's story.
